一、描述

记录一下已经push到服务器端的代码回退方法，撤销操作包括：

  1. 撤销push提交
  2. 撤销push代码
  3. 回退push提交
  1. 回退push代码
  2. 撤销commit提交
  3. 回退commit提交
  4. 撤销add提交
  5. 撤销add代码
  6. 撤销add操作
  7. 回退add代码

二、push回退

2.1.关键词

回退push代码，撤销push提交，撤销push代码，回退已经push的代码，回退push版本文件

撤销推送（git push）需要进行三步操作。

  1. 撤销提交信息
  2. 撤销
  3. 修改代码，重新提交和推送

2.2.撤销提交信息

通过git log查看提交信息。

Author: daizelai <daizelai@gbcom.com.cn>

    新增BUG模板

我们需要撤销到修复通话菜单图标使用的是信息图标的问题这次的提交，所以需要回退到修复通话菜单图标使用的是信息图标的问题的版本，所以回退到的版本号是de2c6d310979d9af074a53c9baca9f84f6f99b76。

然后，通过git reset –soft <版本号>重置至指定版本的提交，达到撤销提交的目的。

还可以选择参数hard，会撤销相应工作区的修改，那样辛辛苦苦写的代码就没有了，一定要谨慎使用。

daizelai@GBCOM-20210524I MINGW64 /d/code/gbcom_gitlab/WZS2_Android_APP (develop)
$ git reset --soft de2c6d310979d9af074a53c9baca9f84f6f99b76

通过git log确认是否撤销成功。

2.3.撤销

通过git push origin master –force强制提交当前版本号，以达到撤销版本号的目的，撤销之前，先看一下git状态。

daizelai@GBCOM-20210524I MINGW64 /d/code/gbcom_gitlab/WZS2_Android_APP (develop)
$ date
Thu Aug 18 09:51:42     2022

daizelai@GBCOM-20210524I MINGW64 /d/code/gbcom_gitlab/WZS2_Android_APP (develop)
$ git status
On branch develop
Your branch is behind 'origin/develop' by 1 commit, and can be fast-forwarded.
  (use "git pull" to update your local branch)

Changes to be committed:
  (use "git restore --staged <file>..." to unstage)
        modified:   app/build.gradle
        modified:   app/src/main/AndroidManifest.xml
        modified:   config.gradle

Untracked files:
  (use "git add <file>..." to include in what will be committed)
        app/src/main/java/com/gbcom/metrosystem/mvvm/broadcast/

上面可以看到，本地已经与gitlab服务器不一样了，git提示我们需要pull，因为我们是撤销，所以接下来直接执行撤销命令就可以了。

daizelai@GBCOM-20210524I MINGW64 /d/code/gbcom_gitlab/WZS2_Android_APP (develop)
$ git push origin develop --force
Total 0 (delta 0), reused 0 (delta 0), pack-reused 0
remote:
remote: To create a merge request for develop, visit:
remote:   http://gitlab.gbcom.com.cn/daizelai/wzs2_android_app/-/merge_requests/new?merge_request%5Bsource_branch%5D=develop
remote:
To gitlab.gbcom.com.cn:daizelai/wzs2_android_app.git
 + 0977407...de2c6d3 develop -> develop (forced update)

daizelai@GBCOM-20210524I MINGW64 /d/code/gbcom_gitlab/WZS2_Android_APP (develop)
$

这时，服务器端也已经撤销了

再来看一下git状态，可以看到，git服务器也撤销成功了。

但是这里是有问题的，这里只是撤销了push，其实当前代码三个modified文件是add状态的，如果要修改文件状态，还需要撤销add，然后修改文件。至于怎么撤销add，可以看下一节。
