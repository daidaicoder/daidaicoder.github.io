一、描述

打tag操作记录。

二、开始

2.1.打新标签

打新标签：git tag <name>，

git tag v2.0

2.2.查看标签

查看标签：

# git tag
v2.0

2.3.打历史标签

如果今天是周五了，需要打周二提交的时候的标签怎么办呢？

方法是找到历史提交的commit id，然后打上就可以了。

daizelai@GBCOM-20210524I MINGW64 /d/code/gbcom_gitlab/SH18_SC60_Android_APP (develop_sc60)
$ git log --graph --pretty=oneline --abbrev-commit
* dd4936b (HEAD -> develop_sc60, origin/develop_sc60) 修改文件说明
* 384060b (origin/master, origin/develop, origin/HEAD, master) 功能优化
* 69357ef (tag: R0105) 增加版本说明
* 08e0ab1 最新版本提交

比如这里的69357ef (tag: R0105) 增加版本说明的提交需要打标签，它对应的commit id是69357ef，可以执行以下命令：

# git tag v1.0 69357ef

2.5.创建带说明的标签

创建标签时还可以使用-a指定标签名，使用-m指定标签说明文字。

git tag -a tagName -m "xxxxxxx"

给某个指定的commit添加tag，-m 后跟备注信息

# git tag -a v3.0 dd4936b -m "带说明的标签"

2.6.标签删除

删除本地标签

如果标签不需要了，或者打错了，也可以删除标签。

git tag -d <tagname>

删除本地标签

# git tag -d v3.0
Delete tag 'v3.0' (was 1f0b51d)

在没有推送到远程前，打错的标签可以在本地安全的删除。

删除远程标签

git push origin --delete v1.0.0
git push origin :refs/tags/<tagname>

2.7.推送标签到远程

推送单个本地标签到远程

git push origin <tagname>

推送：

# git push origin v3.0
Total 0(delta 0), reused 0 (delta 0)
To gitlab.gbcom.com.cn:daizelai/sz14_android_7.1.2_qcom.git
 * [new tag]      v3.0 -> v3.0

推送全部本地标签到远程

git push origin --tags

2.8.搜索tag

# git tag -l 'v3.0*'
