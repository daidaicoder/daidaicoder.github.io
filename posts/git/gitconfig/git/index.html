一、描述

1.1.文档背景

by daizelai on 2022/09/27 17:38

现在一般公司都是使用git进行管理，有可能是gitlab，也有可能是github或gitee等。但是如果多个项目使用不同的托管平台，那么这个时候就需要git多账号切换的功能。

关键字：git多账号配置,git多平台管理,git多用户配置,gitlab多账号配置,github多平台管理,gitee多用户配置。

1.2.需求分析

  • 如果配置了全局git账号设置，需要清除全局配置。
  • 需要不同账号对应不同的git托管平台进行项目维护管理。

3.5.添加公钥到远端仓库

读取取刚刚生成的公钥id_rsa_itcss_gitlab.pub内容，然后把内容添加到itcss|gitlab里面去。

3.6.配置密钥管理文件

为了更方便的管理多个git账号，我们需要一个config的配置文件，用来集中配置。 在.ssh目录下，创建config文件，我这里因为之前使用vs code远程连接创建过，所以直接在里面配置就可以了。

# Read more about SSH config files: https://linux.die.net/man/5/ssh_config
Host dockerUbuntu
    HostName 192.168.13.56
    User root
    Port 10042
Host dockerCentos8
    HostName 192.168.13.56
    User root
    Port 10052
# gbcom.gitlab account
Host gbcom.gitlab.com
HostName gitlab.gbcom.com.cn
User daizelai
IdentitiesOnly yes
PreferredAuthentications publickey
IdentityFile C:\\Users\\daizelai\\.ssh\\id_rsa

# itcss account
Host itcss.cn.gitlab
# HostName 127.0.0.1 配置这个地址也是可以的
HostName 192.168.13.55
User daizelai
IdentitiesOnly yes
PreferredAuthentications publickey
IdentityFile C:\\Users\\daizelai\\.ssh\\id_rsa_itcss_gitlab
Port 9010

配置图解：

Host ---------远端仓库的别名，简写 例如： 配置后 git@gitee.com 可以写为 git@gitee
HostName ---------远端仓库地址或IP（不支持配置端口）
User ---------仓库上的用户名
PreferredAuthentications ---------强制使用Publickey验证
IdentityFile ---------指向仓库私钥的绝对路径
Port ---------远端仓库端口，一般不需要设置，特殊情况除外

3.8.配置项目账号

这个时候虽然可以clone下来，但是如果想要push，还是不行的，需要配置项目账号信息。

主要代码如下：

git config user.name "daizelai"
git config user.email "daizelai@gbcom.com.cn"
