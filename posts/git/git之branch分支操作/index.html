分支常用操作

  • 查看分支：git branch
  • 创建分支：git branch <name>
  • 切换分支：git checkout <name>
  • 创建+切换分支：git checkout -b <name>
  • 在本地创建和远程分支对应的分支：git checkout -b branch-name origin/branch-name
  • 建立本地分支与远程分支的关联：git branch --set-upstream branch-name origin/branch-name

下载指定分支

git clone -b 分支名字 git仓库地址

clone远程代码，注意这里clone的是master分支代码

git clone -b develop_sc60 git@gitlab.gbcom.com.cn:daizelai/sh18_android_app.git

删除本地分支

使用如下命令可以删除本地分支，注意这里-d修改为-D则会执行强制删除操作。

git branch -d develop_sc60

删除远程分支

可以使用git branch -a查看是否删除成功。

## origin远程仓库名称，develop_sc60远程分支名称
git push origin --delete develop_sc60

本地分支重命名

## master ：旧的本地分支名称   newbranchname：新的本地分支名
$ git branch -m develop_sc60 newbranchname

分支修改现场保存

储藏

stash功能，可以把当前工作现场“储藏”起来，等以后恢复现场后继续工作：

git stash

还可以同时添加一个标识。

$ git stash save 'cache-change'
Saved working directory and index state On dev: cache-change

恢复

事情做完以后，接着回到刚刚保存的分支。

git checkout develop
$ git status
On branch develop
nothing to commit, working tree clean

但是是工作区是干净的，之前的工作内容在哪里呢？使用git stash list命令查看。

$ git stash list
stash@{0}: WIP on dev: f52c633 add merge

可以看到工作现场还在，git把stash内容存在某个地方了，恢复一下就可以了。

还可以恢复前先看一下堆栈中最新保存的stash和当前目录的差异。，也就是将stash合并到当前工作区。

# git stash show

apply恢复

使用git stash apply恢复，但是恢复后，stash内容并不会删除，需要使用git stash drop命令来删除。 该命令能够将堆栈的内容多次应用到工作目录中，适应于多个分支的情况。

pop恢复

使用git stash pop恢复，同时把stash内容也删除了。

git stash pop

git别名操作

添加别名。

# git config --global alias.co checkout
# git config --global alias.ci commit
# git config --global alias.br branch

后面就可以简写命令了。

# git ci -m "test info"

合并某分支到当前分支

当前在master分支，合并develop分支到当前master分支。

# git merge develop --no-ff

注意：合并一个没有关联的分支可能会提示fatal: refusing to merge unrelated histories 这时可以在合并是增加一个参数--allow-unrelated-histories

# git merge develop --allow-unrelated-histories
