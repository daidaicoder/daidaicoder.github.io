一、描述

用git命令，可以看到自己之前的操作记录，使用的命令是git log和git reflog，它们的区别是：

  • git log 命令可以显示所有提交过的版本信息
  • git reflog 可以查看所有分支的所有操作记录（包括已经被删除的 commit 记录和 reset 的操作）

二、实践

2.1.查看所有提交的版本信息

daizelai@GBCOM-20210524I MINGW64 /d/code/gbcom_gitlab/SH18_Android_APP (develop_sc60)
$ git log
commit 89d719a3a47551900c3999878cc30435e751db19 (HEAD -> develop_sc60)
Author: daizelai <daizelai@gbcom.com.cn>
Date:   Mon Dec 20 11:30:44 2021 +0800

    日志内容修改

commit 384060b4cd73cc64f85113f2517749d46248e051 (origin/develop_sc60, origin/develop, develop)
Author: daizelai <daizelai@gbcom.com.cn>
Date:   Fri Dec 17 16:17:06 2021 +0800

    功能优化

commit 69357ef262ce82b8d2adaf68dd0a9ce6af6f7be4 (tag: R0105)
Author: daizelai <daizelai@gbcom.com.cn>
Date:   Mon Jun 21 10:26:44 2021 +0800

    增加版本说明

commit 08e0ab1177716bf4b468e7d0311f85cfb12df91b
Author: daizelai <daizelai@gbcom.com.cn>
Date:   Mon Jun 21 10:22:26 2021 +0800

如果觉得显示的太杂，还可以加上参数--pretty=oneline，此时只会显示版本号和提交时的备注信息了。

daizelai@GBCOM-20210524I MINGW64 /d/code/gbcom_gitlab/SH18_Android_APP (develop_sc60)
$ git log --pretty=oneline
89d719a3a47551900c3999878cc30435e751db19 (HEAD -> develop_sc60) 日志内容修改
384060b4cd73cc64f85113f2517749d46248e051 (origin/develop_sc60, origin/develop, develop) 功能优化
69357ef262ce82b8d2adaf68dd0a9ce6af6f7be4 (tag: R0105) 增加版本说明
08e0ab1177716bf4b468e7d0311f85cfb12df91b 最新版本提交

2.2.查看所有分支的所有操作记录

使用git reflog命令可以查看所有分支的所有操作记录，包括已经被删除的commit记录和reset的操作。

说明：

  1. 最左侧黄色字体列为修改的commit的前7位，根据7位可以将代码恢复到对应节点位置。
  2. {n}表示HEAD更改历史记录，最近的操作在上面。
  3. 通过HEAD{n}语法可以引用存在reflog中的提交。
  4. 与HEAD~n功能类似，但HEAD{n}引用的是reflog记录中的commit提交，而HEAD~n引用是commit提交历史记录。
  5. rebase commit等为git操作与提交节点相关的具体指令。
  6. 最后一个冒号后面的字串为提交信息的摘要信息。
