应用master分支的修改到develop分支

描述

在master分支上修复了bug后，evelop分支最早期也是从master分支上分出来的，所以develop分支上也存在相同的bug。

那怎么在develop分支上修复同样的bug呢？重复修改代码一次然后提交？当然可以，但是有没有更便捷的方法呢？

同样的bug，在develop分支上修复，只需要把5c253d6 fix bug loginbug提交所做的修改“复制”到develop分支，注意这里并不是把整个master分支都merge过来。

复制某次提交到另一分支

git提供了一个cherry-pick命令，让我们能复制一个特定的提交到当前分支。

# git branch
* develop
  master
# git cherry-pick 5c253d6
[master 2d4a991] fix bug login
  1 file changed, 1 insertion(+), 1 deletion(-)
