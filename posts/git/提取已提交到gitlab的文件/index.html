二、如何只提取Git某次提交修改过的文件

zelai未经过验证。

笨办法： 首先，保证你的本地repo处于最新状态（该提交的提交了，该pull都pull好了，等等） 用git reflog找到对应的commit 用git reset –hard HEAD@{id}回退到指定的commit 复制需要的文件 用git pull origin master回到最新的状态

三、提取已提交的gitlab的文件

3.1.提取发布某一版本的所有文件

这是把所有版本文件都提取出来，如Android7.1.2项目所有文件。

git archive 4ff3df661993e4839034c4e855532680eaeee0d8 | gzip > package1.tar.gz

3.2.git打包某一次提交的所有文件

git diff --name-only f1bf04360aeca36ba861cb88c2c26714ba865765 4ff3df661993e4839034c4e855532680eaeee0d8 | xargs tar -jcvf bidding2.tar.bz2

359d311b06ff2d79ba535aa67ed26fee2021110d 是某次提交之前的commit id,241f5cd9c6b1f31a60f6d208b28d485adf51ebdf 是某次提交的commit id, bidding.tar.bz2是某次提交所有文件的打包。

zelai使用的文件打包。

[android_SC20@localhost android_7.1.2_qcom]$ git diff --name-only f1bf04360aeca36ba861cb88c2c26714ba865765 4ff3df661993e4839034c4e855532680eaeee0d8 | xargs tar -jcvf bidding2.tar.bz2
device/qcom/common/base.mk
packages/apps/subwaysystem/Android.mk
packages/apps/subwaysystem/subwaysystem.apk
[android_SC20@localhost android_7.1.2_qcom]$

3.3.打包某一次提交的所有文件

zelai提示，这个好。

[android_SC20@localhost android_7.1.2_qcom]$ git diff-tree -r --no-commit-id --name-only 4ff3df661993e4839034c4e855532680eaeee0d8| xargs tar -rf mytarfile.tar
[android_SC20@localhost android_7.1.2_qcom]$

3.4.获得两个修订版本之间更改的文件列表

这很简单：

git diff -–name-only commit1 commit2 > /path/to/my/file
