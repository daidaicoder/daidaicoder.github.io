一、查看设备情况

1.1.查看AP

先来看一下网关和网关下的AP已正常上线。

[root@CGW3000T ~]# gctl showap
AP 0001: ip=172.16.66.91 mac=ac:34:cb:38:db:c0 model CA2025C name Access Point
AP 0002: ip=10.60.10.158 mac=28:51:32:10:91:e2 model CGW3000T name CGW3000T

AP_nums = 2, RegisterApNum = 0, OfflineApNum = 0

1.2.查看网关的版本

使用vlist命令可以查看系统版本情况。

[root@CGW3000T ~]# vlist
 Version  : 1.3.3.3
 Revision : 5403
 BuildTime: 2016-03-03 10:53:07

三、云平台配置

因为是厂园同事范晨熙帮忙配置的，这里就省略了，不过应该和我们云平台的配置是差不多的。

这里需要注意一个问题，我一开始把网关当成我们云平台了，一挂上去，我本地的平台就死翘翘了。厂园的网关和我们这边的网关还是有差别的。

四、获取SSID

4.1.关于AP的模式

如果是胖AP，你直接先在AP的WEB界面上配置好SSID，然后挂在网关下就好了呢。如果是瘦AP，那就在LAC上上线吧。

如果是瘦AP就和AP+AC的配置是一样操作的，可以向测试部周燕他们请教一下怎么配置吧，我这里是胖模式，先不管了。

~ # cat /tmp/ap_work_mode 
ap_work_mode=fat // 胖模式

4.2.查看SSID

从网关上面通过telnet方式连接到AP上去，注意用户名和密码都是admin哦。

4.3.修改SSID名称

默认的AP的SSID名称，有可能会有重复，导致连接冲突，这里可以修改一下SSID的名称，需要注意的是，这样修改一重启就会没有了，回到默认的。

~ # iwconfig ap0_0 ESSID lai_edu_ap
~ # iwconfig ap0_0
ap0_0     IEEE 802.11ng  ESSID:"lai_edu_ap"  
          Mode:Master  Frequency:2.412 GHz  Access Point: 00:34:CB:38:DB:C0   
          Bit Rate:144.4 Mb/s   Tx-Power=24 dBm   
          RTS thr=2346 B   Fragment thr:off
          Encryption key:off
          Power Management:off
