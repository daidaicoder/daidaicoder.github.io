一、CGW500网关抓包

1.使用tftp连接网关

在电脑和网关间进行文件传输，这里使用tftp工具，这里的网关为CGW500，IP地址为10.60.0.37，注意一点，哪里打开运行了tftp.exe，那么这里就是服务器，比如这里是在10.60.0.95上面打开的，10.60.0.95本地电脑就是服务器。

2.切换到tmp目录下

因为CGW500只有/tmp目录才能读写保存，所以切换到该目录下。

# cd /tmp

3.把tcpdump导入网关

Tftp导入tcpdump，语法格式如下：

# tftp –gr tcpdump xx.xx.xx.xx

从tftp服务器端也就是本地电脑（10.60.0.95）上传文件tcpdump到网关（10.60.0.37）

/tmp # tftp -gr tcpdump 10.60.0.95        
/tmp # ls -lh tcpdump 
-rw-r--r--    1 root     root         2.1M Oct 14 16:45 tcpdump

4.修改权限

/tmp # chmod u+x tcpdump

5.抓包

抓包语法格式：

/tmp # ./tcpdump –i br0 –w pingan.cap

查看要抓的SSID所在网口名称

/tmp # iwconfig
ap0_5     IEEE 802.11ng  ESSID:"00000003"

抓包

/tmp # ./tcpdump -i ap0_5 -w pingan.cap
tcpdump: WARNING: ap0_5: no IPv4 address assigned
tcpdump: listening on ap0_5, link-type EN10MB (Ethernet), capture size 65535 bytes
^C339 packets captured
339 packets received by filter
0 packets dropped by kernel

提示： 如果想判断报文有没有发出去，比如平安APP发送的数据到test上面去，可以抓一下ech0的包，看看有没有数据，当然这个eth0也要根据情况得到的。

6.下载文件到本地

抓包完成后，通过tftp把网关中的cap文件拿到本地来。

语法格式：

# tftp –pl pingan.cap xx.xx.xx.xx(ip地址)上传到本地

下载文件电脑（tftp服务器端）

/tmp # tftp -pl pingan.cap 10.60.0.95

二、Windows上的sftp命令

在Windows上开启sftp服务器tftpd32.exe, 然后选定一个默认的目录作为服务器根目录，此时，tftpd32.exe自动读取pc的ip, 该ip就是tftp服务器的ip, 比如10.60.0.95

同时windows上面的cmd中是有tftp命令的，cd到一个目录作为客户端目录。

命令1：

# tftp 10.60.0.95 put index.php

该命令可以把tftp客户端目录下的index.php文件上传到tftp服务端，其中的10.60.0.95是服务端ip, put表示上传。

命令2:

# tftp 10.60.0.95 get test.txt

该命令可以把tftp服务端目录下的test.txt文件下载到tftp客户端， 其中的10.60.0.95是服务端ip, get表示下载。

三、Wireshark分析报文

3.1.场景描述

手机终端连接我本地CGW500网关下名为 0000003的SSID，打开安APP，点击一键自动连接后，平安APP会向认证平台(test)发送认证数据，发送的认证的址为一开始给平安的平台认证地址。

3.2.Wireshark打开pingan.cap

打开Wireshark后，打开报文，因为平安APP提交的数据是到test上面去，test的IP地址为115.29.9.167。 可以通过下面这样过滤一下报文。

tcp && ip.dst==115.29.9.167

3.3.报文分析

1.网关抓包之手机连接SSID访问云平台报文查看 可以看到Hypertext Transfer Protocol中的报文，源地址、提交地址、POST方式提交等信息。

2.网关抓包之post数据查看 这里是通过post提交的json格式数据，下面的报文可以看出类似键值对的数据，说明报文已经拿到了。
