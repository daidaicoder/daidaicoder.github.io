一、apache配置localhost虚拟主机步骤

1.1.配置httpd

打开apache目录下httpd文件(如：C:\apache\Apache2.2.17\conf\httpd.conf)，找到如下模块：

#Virtual hosts
#Include conf/extra/httpd-vhosts.conf

去掉前面的#，这样就开启了httpd-vhosts虚拟主机文件。

1.2.httpd-vhosts

打开httpd-vhosts文件(如：C:\apache\Apache2.2.17\conf\extra\httpd-vhosts.conf)，配置好localhost虚拟主机，参照httpd-vhosts文件中实例，修改成如下：

<VirtualHost *:80>
    ServerAdmin webmaster@dummy-host.hleclerc-PC.ingenidev
    DocumentRoot "C:/www/web"
    ServerName www.test.com
    ServerAlias www.dummy-host.hleclerc-PC.ingenidev
    ErrorLog "logs/dummy-host.hleclerc-PC.ingenidev-error.log"
    CustomLog "logs/dummy-host.hleclerc-PC.ingenidev-access.log" common
    <Directory C:\www\web>
        Options FollowSymLinks
        AllowOverride None
        Order deny,allow
        Allow from all
    </Directory>
</VirtualHost>

修改配置如下：

  • DocumentRoot 修改为本地wamp环境下的www目录(如：C:\www\web)
  • ServerName 改为localhost
  • 重启Apache，这样就配置好了，本地可以用www.test.com来访问C:/www/web 这个站点。

注意事项：

  1. httpd文件，打开Include conf/extra/httpd-vhosts.conf模块
  2. 目录路径，如 C:/wamp/www/a
  3. 访问权限，如上Deny from all修改为Allow from all
  4. hosts文件，配置虚拟域名host指向
  5. httpd-vhosts文件，配置虚拟主机

二、在windows下面配置apache虚拟主机

2.1.配置虚拟主机需要3个文件

  1. Apache/conf/httpd.conf
  2. Apache/conf/extra/httpd-vhosts.conf (这个地版本的apache可能没有，可自己创建，也可以不要此文件，而将配置直接写在httpd.conf里面，写在extra/httpd-vhosts.conf只是为了管理方便，不让httpd.conf文件内容很多而已)。
  3. C:\WINDOWS\system32\drivers\etc\hosts

2.2.配置

2.1.httpd.conf 找到VirtualHost example，在后面引入httpd-vhosts.conf文件：

Include conf/extra/httpd-vhosts.conf

这个句话，高版本的apache里面已经写有了，只需吧Include前面的#号去掉就OK

2.2.httpd-vhosts.conf配置文件的内容如下：

#
# DocumentRoot 是网站文件存放的根目录
# ServerName 是网站域名, 需要跟 DNS 指向的域名一致
#

####################### www.itcss.cn ####################
#VirtualHost *:80#
    ServerAdmin webmaster@iwebshop.com
    DocumentRoot "e:/itcss"
    ServerName www.itcss.cn
    ErrorLog "logs/www.itcss.cn-error.log"
    CustomLog "logs/www.itcss.cn-access.log" common
#/VirtualHost#

#Directory "e:\itcss"#
    Options Indexes FollowSymLinks
    AllowOverride All
    Order allow,deny
    Allow from all
#/Directory#
